## 1. Почему мы провели DPIA

Проведение DPIA **обязательно** по ст. 35 GDPR, когда срабатывает
один или несколько следующих триггеров.  Для Vocaband применяются
**три независимых триггера**:

| Триггер (ст. 35(3) GDPR + Рекомендации EDPB 04/2017) | Релевантность для Vocaband |
|---|---|
| Систематическая обработка персональных данных детей | Основная база пользователей — учащиеся 9–15 лет. |
| Использование инновационных технологий (AI / ML на персональных данных) | OCR (Gemini), генерация предложений (Claude), конструктор уроков, генератор экзамена Bagrut. |
| Крупномасштабная обработка | Разработана для целых образовательных округов Израиля; целевой масштаб >10 000 учащихся на округ. |

DPIA также требуется согласно **Положению 9 израильских Положений
о защите частной жизни (информационная безопасность) 5777-2017**
для любой базы данных «среднего» или «высокого» уровня безопасности,
обрабатывающей данные несовершеннолетних — что соответствует
Vocaband.

## 3. Необходимость и пропорциональность

Обработка **необходима**, потому что:

- Изучение лексики требует отслеживания прогресса по каждому
  учащемуся; анонимный электронный журнал противоречил бы
  образовательной цели.
- Функции ИИ (OCR, генерация предложений) необходимы для
  сокращения времени подготовки учителя — иначе учителя не
  использовали бы платформу.
- Live-соревнования в классе требуют состояния в реальном
  времени по каждому учащемуся.

Обработка **пропорциональна**, потому что:

- Учётные записи учащихся по умолчанию не собирают **электронную
  почту**, **реальное имя**, **государственное удостоверение
  личности**, **фотографию**, **местоположение**.  См. `RoPA.md →
  Activity 1`.
- Функции ИИ обрабатывают **только слова** — персональные данные
  не отправляются в Anthropic / Gemini OCR / Gemini Translate.
  Полезные нагрузки OCR удаляются на стороне поставщика ИИ
  после возврата API.
- Состояние live-соревнований **только в памяти**, никогда не
  записывается на диск (за исключением финальной строки баллов,
  если учитель подтверждает в конце игры).
- Поведенческая реклама и маркетинг **запрещены** инженерными
  гарантиями Vocaband (`docs/legal/DPA-ru.md §2.3`).

## 4. Анализ рисков (на высоком уровне)

Полный реестр рисков: `docs/RISK-REGISTER.md` (279 строк, Положение
6 Положений 2017).  Пять остаточных рисков с наибольшим влиянием
после применения средств управления, описанных в §5:

| # | Риск | Вероятность (остаточная) | Серьёзность (остаточная) | Краткое описание митигации |
|---|---|---|---|---|
| R-1 | Скомпрометированная учётная запись учителя → межклассовый просмотр данных | Возможно | Средняя | План MFA (пункт H-1 аудита); RLS изолирует по классу; аудит-лог захватывает массовые чтения. |
| R-2 | Утечка ключа service-role → полный дамп БД | Маловероятно | Критическая | Ключ хранится только в Fly secrets; `INCIDENT-RESPONSE.md` содержит сценарий «Компрометация ключа service-role»; ключ отсутствует в клиентском бандле. |
| R-3 | Загрузка OCR случайно содержит PII учащегося | Возможно | Низкая | Функция только для учителя; UI загрузки не рекомендует загружать работы учащихся; поставщик в режиме нулевого сохранения; изображение удаляется после извлечения. |
| R-4 | Сбой Supabase в школьные часы → потеря учебной сессии | Маловероятно | Средняя | Multi-AZ в Франкфурте; ежедневные резервные копии (≤30 дней) + R2 вне площадки (≤365 дней); непрерывность бизнеса в `INCIDENT-RESPONSE.md`. |
| R-5 | Утечка у субпроцессора (Anthropic / Google / и т.д.) | Маловероятно | Средняя | Контракты DPF + нулевое сохранение; TIA по поставщику в `SUBPROCESSORS.md`; сценарий цепочки поставок в `INCIDENT-RESPONSE.md`. |

Ни один риск не оценивается как **высокая вероятность × критическая
серьёзность** (порог «остановки линии» согласно рубрике реестра
рисков).

## 5. Митигации и средства управления безопасностью (краткое изложение)

| Область | Средства управления |
|---|---|
| **Шифрование** | TLS 1.3 при передаче; эквивалент AES-256 при хранении (Supabase). |
| **Аутентификация** | Google OAuth (сертифицирован DPF) или электронная почта + OTP для учителей; код класса + PIN или анонимная сессия для учащихся.  MFA оператора в консолях Supabase / Fly / Cloudflare. |
| **Авторизация** | Row-Level Security на каждой таблице с PII; чтение, ограниченное арендатором (по классам), обеспечивается на уровне БД. |
| **Аудит-лог** | `audit_log` только для добавления с триггерами уровня БД, запрещающими модификацию (`20260518120000_audit_log_immutability.sql`).  Сохранение 730 дней. |
| **Очистка PII** | `scrubPii.ts` работает в `Sentry.beforeSend` И на каждом вызове `console.*` через `installScrubbingConsole`. |
| **Согласие** | По умолчанию выключенные неосновные cookies, симметричные кнопки «Принять/Отклонить»; Sentry заблокирован за согласием `analytics: true`. |
| **Права субъектов** | В приложении: `export_my_data` (ст. 15/20) и `delete_my_account` (ст. 17), покрывающие все таблицы пользовательских данных, включая auth.users. |
| **Управление субпроцессорами** | Публичный реестр `docs/SUBPROCESSORS.md` с TIA по поставщику + статус DPF; подписка на уведомление ≥30 дней. |
| **Реагирование на инциденты** | `docs/INCIDENT-RESPONSE.md` — уведомление DPO в течение 24 ч, уведомление PPA в течение 72 ч, уведомление отдела информационной безопасности Министерства образования, сценарии цепочки поставок + компрометации ключа, ежеквартальный настольный тренинг. |
| **Тестирование на проникновение** | `scripts/security-pen-test.sh` (набор тестов RLS + заголовков + неизменности аудит-лога). |

## 6. Данные детей — дополнительные соображения

- **Не требуется реальное имя.**  Учащиеся выбирают отображаемое
  имя — приветствуются первые имена и прозвища.
- **Нет электронной почты.**  Учётные записи учащихся по умолчанию
  используют синтетический внутренний адрес; никакое сообщение
  никогда не доставляется в почтовый ящик учащегося.
- **Нет публичных профилей.**  Прогресс виден только в собственном
  классе учащегося, классному руководителю и школьному
  администратору, если назначен.
- **Права родителей.**  В настоящее время реализуются по
  электронной почте на адрес `privacy@vocaband.com`.  Прямой
  канал внутри продукта ожидается в рамках пункта C-1+C-8 аудита.
- **Безопасность ИИ.**  Текст, создаваемый ИИ и потребляемый
  несовершеннолетними, работает с фильтрами безопасности,
  соответствующими опубликованным рекомендациям по безопасности
  детей субпроцессоров ИИ; никакой свободный чат, создаваемый
  ИИ, никогда не показывается напрямую учащемуся без
  посредничества учителя.
