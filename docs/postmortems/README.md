# Post-mortems

> Seeded so on-call doesn't have to `mkdir` under pressure.  See
> `docs/INCIDENT-RESPONSE.md → "Post-incident — within 7 days"` for
> the template.

One file per incident, named `<YYYY-MM-DD>-<short-name>.md` to match
the corresponding `incidents/<same-name>/` evidence directory.

Read at the next team checkpoint after the incident.  Action items
get tracked in `docs/open-issues.md` or the issue tracker.

Do **not** name individuals in a way that attributes blame — focus
on the failed control or process gap, not the person who hit the bug.
